# 蛋白质设计超级工厂 — 架构计划

> **目标：** 利用DeepSeek API优惠期，构建一个能消耗数十亿token的蛋白质设计知识引擎。不是简单的查看器+聊天，而是一个**持续运行的批处理知识工厂**——后台队列自动生成结构化知识，前端3D查看器作为交互入口。所有产出存入Obsidian知识库，永久复用。

**生成策略：**
- 每章分3-5节，每节独立生成（利用长上下文优势一次生成完整节）
- 每节要求：数学推导 + 代码示例 + PDB实战 + 自测题
- 章节间交叉引用（"详见3.2节 Rama图分析"）
- 生成后自动存入Obsidian知识库

**Token 消耗估算：**
- 约100个独立文件（章/节）
- 每节平均输出3000-8000字 ≈ 5K-15K tokens 输出
- v4-pro 思考链约为输出的 5-10x
- 每节总消耗：30K-150K tokens
- **教材引擎总消耗：3M-15M tokens** ← 不够，需要加深

**加强版生成策略（消耗最大化）：**
- 每章不是一次生成，而是迭代精修：
  1. 初稿生成 (30K tokens)
  2. 自检审查 + 补充缺失细节 (20K tokens)
  3. 习题+答案生成 (15K tokens)
  4. 代码实战演练 (10K tokens)
  5. 交叉引用标注 (10K tokens)
- 每章5轮迭代 × 85K = 425K tokens
- 100章 × 425K = **42.5M tokens**

还可以继续扩大——如果每章加上"文献支撑"（引用关键论文并生成摘要）、"历史背景"、"常见误区"等扩展内容：
- 扩展到150章 × 500K = **75M tokens**
- 加上课后项目（每章一个mini-project，10K-50K tokens）
- 总教材部分：**~100M tokens**

### 引擎3：突变宇宙扫描器 🔬

**定位：** 对蛋白质做系统性虚拟突变扫描——所有位置 × 所有20种氨基酸 + 深度学习分析。

**单个蛋白质的扫描矩阵：**
```
GFP: 238个残基位置 × 19种突变体（每个位置20种aa，减野生型）
= 4,522 个单点突变体

对每个突变体，生成分析报告：
├── 位置上下文（二级结构/溶剂暴露/保守性）
├── ESM打分（野生型 vs 突变型 log-likelihood 差异）
├── 侧链体积变化（Å³）
├── 电荷变化（如果相关）
├── 氢键网络影响预测
├── 疏水核心扰动分析
├── 空间冲突检测（新侧链与周围残基的范德华重叠）
├── 文献中类似突变的已知效应
├── AI综合评估（用v4-pro深度推理）:
│   ├── 可能的功能影响（荧光强度/稳定性/折叠）
│   ├── 突变推荐等级（1-5星）
│   ├── 组合突变建议（"如果同时突变X和Y..."）
│   └── 解释为什么（化学直觉+计算证据）
└── 实验优先级排序
```

**Token 消耗估算（单个蛋白质）：**
- 4,522个突变体
- 每个突变报告输出约500-1000字 ≈ 1K-2K tokens输出
- 用v4-flash做批量打分（速度快、token消耗小）
  - flash评分：每个突变体 ~2K tokens（输入+输出）
  - 4,522 × 2K = **9M tokens**
- 选前200个最有趣突变，用v4-pro深度分析
  - pro深度分析：每个 ~15K tokens
  - 200 × 15K = **3M tokens**
- 汇总报告：排序 + 组合建议 = **2M tokens**
- **单个蛋白质 ≈ 14M tokens**
- 计划扫描50个蛋白质 = **700M tokens**

**还可以更暴力：**
- 加上双突变组合扫描：对关键区域做 N×M 组合
- 一条10残基的环区 = 100个双突变 × 50K tokens each = 5M tokens
- 50个蛋白质 × 10个热点区域 × 5M = **额外2,500M tokens**

### 引擎4：交叉引用引擎 🔗

**定位：** 在蛋白质之间建立联系，发现设计规律。

**功能：**
```
1. 同源结构对比
   - 自动下载同源PDB结构（如果有网络访问）
   - 结构叠合 + RMSD分析
   - 序列保守性映射到3D结构
   - 功能位点比较
   
2. 折叠类型分类
   - 识别折叠类型（TIM桶/罗斯曼折叠/免疫球蛋白折叠等）
   - 同折叠类型蛋白质的系统比较

3. 设计规律提取
   - "在所有荧光蛋白中，发光核心周围5Å内的残基有什么共同特征？"
   - "成功的热稳定性突变通常出现在什么位置？"
   - AI自动生成设计规则和最佳实践
```

**Token 消耗：** 约100M-300M tokens（取决于覆盖范围）

### 引擎5：AI导师（交互前端）👨‍🏫

**定位：** 前端实时交互，每次点击、每次对话都消耗token。

**导师行为模式：**
- 用户点击残基 → 自动生成一段解释（200-500字 = 1K-3K tokens/次）
- 用户提问 → 深度回答（500-2000字 = 3K-10K tokens/次）
- AI主动提问：每5-10轮对话，AI反问用户一个引导性问题
- 场景感知：知道你当前在看什么、选了什么、之前讨论过什么

**Token消耗（按使用量）：**
- 假设每天2小时深度教学交互
- 每小时约50轮对话
- 每轮平均5K tokens（输入+输出+思考）
- 每天：50轮 × 2小时 × 5K = 500K tokens
- 优惠期假设90天 = **45M tokens**
- 如果高强度使用：**100M-200M tokens**

## 六、关键问题与决策

1. **ESM模型部署：** 你的bio_project已经有esm_score.py使用facebook/esm。但ESM-2 650M需要约2.5GB VRAM。可以在WSL内用CPU跑（慢但可行），或用DeepSeek API替代打分（更快但更耗token）。建议：**用DS API做打分**（正好消耗token）。

2. **PDB自动下载：** 从RCSB下载需要网络。GFW下 `files.rcsb.org` 通常可访问。备选：预下载一批常用PDB。

3. **前端部署：** FastAPI serve静态文件，WSL内运行，Windows浏览器访问 `localhost:8899`。无需Nginx。

4. **持久化：** 所有知识产出写Obsidian vault (`/host/ai_agents/SecondBrain/protein-design/`)。前端对话仅在会话内保留（避免数据库依赖）。

## 七、为什么这个方案能"烧掉"几十亿token

1. **残基级别深度分析** — 不是"这个蛋白质是GFP"，而是对238个残基逐一写500-2000字的深度档案
2. **全突变空间扫描** — 每个蛋白质4522个单点突变，每个生成分析报告
3. **迭代精修** — 每章教材不是一次生成，而是5轮迭代
4. **v4-pro思考链** — 每个请求的思考链token是输出的5-10倍（token消耗翻5-10倍）
5. **批量作业** — 不是等用户交互，而是后台队列自动跑，24/7持续消耗
6. **交叉引用** — 蛋白质之间比较、模式发现，每对比一对就是一次深度分析
