# Protein Design Studio — 实施计划 v5 (Final)

> **核心决策：Phase 1 不做ProTherm。用诚实的"ESM打分+透明不确定性声明"替代虚假校准。Phase 2 做私有校准：用户输入自己的实验数据，工具越用越准。**
>
> **运行环境：仅支持 WSL2 / Linux。Windows裸机不保证可用（fair-esm依赖编译工具链）。**

---

## Phase 1 目标（12-14天）

**用户场景：** 加载PDB → 浏览3D结构 → 点击残基/批量扫描 → 看到ESM对20种氨基酸的打分 → AI解释结果 → 导出突变列表CSV。

**不做的事：**
- ❌ ProTherm / 任何外部数据库
- ❌ 精确置信区间（改为文字声明）
- ❌ 多着色方案（只保留链着色和B-factor两种）
- ❌ PDF导出（CSV + 剪贴板即可）
- ❌ 移动端适配
- ❌ AI-POOL双Provider（只留OpenRouter）
- ❌ GFP家族知识综合

## 关键设计决策

### 1. Windows 策略

```
声明: 本工具仅支持 WSL2 / Linux。

原因: fair-esm 在 Windows 裸机上需要 MSVC 编译工具链,
      torch-scatter 等依赖编译失败率高。
      WSL2 天然 Linux 环境, 且 CUDA 直通可用。

用户如果坚持 Windows: 提供 Dockerfile (Phase 2)。
```

### 2. 表面残基定义

```
默认: relSASA > 25%
计算: Biopython SASA模块 (Shrake-Rupley算法)
可调: API参数 ?sasa_threshold=0.20

relSASA = SASA(残基在蛋白中) / SASA(同残基在Gly-X-Gly三肽中)
>25%: 表面暴露, 可设计
<25%: 埋藏, 突变风险高 (标注警告)
```

### 3. 不确定性声明

```
前端每个预测旁显示:

  ESM打分: -0.45 (负值 = 可能稳定化)

  ┌─────────────────────────────────────────┐
  │ ⚠️ 重要说明                              │
  │                                          │
  │ ESM-2 650M 预测突变效应的整体相关性约    │
  │ 0.4-0.6 (中等)。此分数未经实验数据校准， │
  │ 仅供筛选参考，不能替代实验验证。          │
  │                                          │
  │ 建议: 先做丙氨酸扫描验证关键位置，        │
  │       再对 top 候选做详细表征。           │
  └─────────────────────────────────────────┘
```

### 4. 私有校准数学（Phase 2 sketch）

```
方向: 简单线性缩放
  用户输入 N 个实验 (ΔΔG_exp, ESM_score) 对
  → 拟合 ΔΔG = a × ESM_score + b
  → 对剩余位置: calibrated_score = a × ESM_score + b
  → 标注: "用 N 个实验点校准, R² = X"
  
  10个数据点: 足够拟合可靠斜率+截距
  3-5个: 全局偏移(b)可用, 斜率不可靠 → 标注
  <3个: 不做校准 → 告知用户需要更多数据
```

## 已知风险与缓解

| 风险 | 概率 | 缓解 |
|------|------|------|
| fair-esm pip安装失败 | 中 | download_esm.py备用; --offline模式; README写明依赖 |
| ESM模型自动下载被墙 | 高 | 预下载脚本; 提示手动下载; 百度网盘备用链接 |
| 3Dmol.js点击映射不准 | 中 | 取最近原子→残基; 多链PDB测试 |
| GPU OOM | 低 | 显存检查; max_batch_size限制; CPU fallback |
| 位置偏移bug (PDB序号≠ESM索引) | 高(已知经典bug) | 单独测试; offset断言; regression test |
