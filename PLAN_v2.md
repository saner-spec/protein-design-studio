> **定位：填补计算预测与实验现实之间鸿沟的普世工具。**

## 一、领域缺口分析：为什么要做这个

### 现有工具的致命缺陷

| 工具 | 能做什么 | 不能做什么 |
|------|---------|-----------|
| Rosetta | 物理能量函数，高精度 | Linux集群，学习曲线极高，没有实验指导 |
| FoldX | 经验力场，有GUI | 单方法，误差大，无文献支撑 |
| PROSS | 自动稳定性设计 | 只做稳定性，不管功能，黑箱 |
| ESM/AlphaFold | 序列打分，结构预测 | 不连接实验数据，不解释"为什么" |
| PyMOL | 3D可视化 | 不是设计工具 |
| **所有工具共通的致命缺陷** | | **没人告诉你预测的置信度** |

### 核心缺口：缺少"置信度层"

```
现有工具:  "GLU222→ALA，ΔΔG = -1.8"  ← 这个数字可靠吗？
          用户不知道。可能只有50%准确率。
          
本工具:    "ΔΔG = -1.8 ± 0.9 kcal/mol
           基于ProTherm中342个类似突变（大→小侧链，表面暴露位）
           校准后Pearson r=0.68。有中等可信度。
           建议：先做3个低成本验证突变再批量合成。"
```

### 为什么现有工具没做这件事

因为做置信度层需要：
1. 解析数万个实验数据点并标准化
2. 对每个数据点计算结构特征
3. 对多种计算方法做系统校准
4. 建立上下文相关的不确定性模型

——这是大量的扎实工作。以前只能靠博士生手动做。现在可以用AI辅助自动化。

- ESMFold API（Meta提供，免费）
- ColabFold（Google Colab，需要Google账号 → 不可用）

## 三、精确Token消耗计算

### 核心原则：每个token都有明确的科学产出

#### 模块A：实验数据库引擎（最高优先级，10天冲刺主力）

**A1. ProTherm 数据库解析与验证**
```
任务：解析ProTherm数据库的~30,000条实验突变数据
每条的AI工作：
  1. 验证PDB ID有效性 + UniProt交叉引用 (输入1K+输出500 = 1.5K)
  2. 标准化突变表示法 (3字母→1字母，野生型序号校准) (1K+500 = 1.5K)  
  3. 分类结构环境 (核心/表面/界面/活性位) (2K+1K = 3K)
  4. 标记潜在错误 (ΔΔG符号矛盾、重复条目) (1K+500 = 1.5K)
  5. 生成自然语言摘要 (1K+800 = 1.8K)
小计: ~9.3K tokens/条 × 30,000条 = 279M tokens
实际：前5000条用v4-pro仔细做，后25000条用v4-flash批处理
加权: 5000×25K(pro) + 25000×6K(flash) = 125M + 150M = 275M tokens
```

**A2. SKEMPI 数据库解析**
```
~7,000条蛋白-蛋白结合突变数据
每条约8K tokens × 7,000 = 56M tokens
```

**A3. ThermoMutDB 解析**
```
~15,000条热力学参数
每条约5K tokens × 15,000 = 75M tokens
```

**A4. 数据库交叉验证与去重**
```
三库合并，检测冲突/重复，生成统一Schema
约30M tokens
```

**数据库引擎小计：275 + 56 + 75 + 30 = ~436M tokens**

#### 模块B：校准与基准系统

**B1. 方法学基准测试**
```
对ESM-2、FoldX、Rosetta（如果可运行）在ProTherm子集上做系统校准：
  - 按结构环境分层（核心/表面/界面）
  - 按突变类型分层（大→小/小→大/极性→非极性/带电变化）
  - 计算各层Pearson r、MAE、RMSE
  - 生成校准曲线
AI工作：对每个方法×每个分层×统计分析 = 约200次深度分析
每次20K tokens × 200 = 4M tokens
但更重要的是——AI生成校准报告的"解读"和"使用指南"
约总50M tokens
```

**B2. 置信度模型**
```
基于校准结果，AI帮助建立：
  - 什么情况下预测可靠（Pearson r>0.7）
  - 什么情况下不可靠（r<0.3）
  - 如何组合多方法提高可靠性
约30M tokens
```

**校准系统小计：~80M tokens**
